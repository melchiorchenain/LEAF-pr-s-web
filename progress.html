Progress

Sign-Up System

To begin LEAF, I devised a sign-up system. To do that, I created a PHP file called inscription.php and began generating a registration form in HTML. The form asks users for their last name, first name, email, password, password confirmation, and profile picture. If the user fills in everything correctly and their email does not already exist in the system, they will be registered. The form does not work if a box is empty or if the two passwords are not identical.

Login System

After making the registration / sign-up system, I built a login system that uses the information stored in the database, which I will develop. Indeed, to log in, I coded a form in HTML just like the sign-up form, but this time it just asks for the already-registered user to enter their email and password. They can then sign in with their information. If the information they shared is incorrect when they log in, a message appears below the login button to alert them.

Registration Database

I developed a registration database after coding the sign-in and login systems. The registration database that I titled User is structured to facilitate the storage, retrieval, modification, and deletion of data. It contains the user's name, last name, email, and encrypted password. The database has a final column that contains an automatically generated user ID. This ID is key to helping me build the messaging system and all the other features that track user activity.

Connecting code to Database

I created a PHP file to connect my PHP code to my database, a quick but crucial step. If the connection fails, the whole media does not function. I, therefore, created a connection PHP file in which I indicated the username of my SQL files, the password to access the database, and my database's name. I also need to inform my host's name. The code depends on the host to connect to the database.

Discussion Forum

I then created the first part of what will become my website's future feed. I coded a discussion forum to allow anyone signed in to LEAF to send messages. To do that, I began by coding a form, like with the sign-up and login parts. Here, however, the form only requires the user to type the message they want to post. After that, I coded (in the same file) the system linked to the future message database. The objective here is that, as the user is signed in when they send the message, LEAF will utilize the user database to post their message along with their name, last name, and profile picture. To do that, I will apply the user ID, which allows me to identify which user sent the message. The algorithm then automatically identifies their name, last name, and profile picture and posts it with the code.

Login Page New Design

When I first coded the user interface for my leaf project, I did not concentrate on the quality of the design or try to make the interface more interactive. The reason I chose to do that because I only used the frontend interface to test the work I was doing in the backend and in this case, make sure that the sign up system and login systems were working in the way I wanted. So I decided to dive int o the design off the login interface now that all the backend interface for the login system is finish. To do an interactive and clean interface that fits with LEAF's design goals, I made a toggle menu that proposes eather a Login or a Signup button. This allows all users to choose easely choose without searching what they want to do when they first arrive on the media. To do that, I linked the code to a javascript file that analyses the buttons being prepessed and makes the propper toggle appear on the page. I then added my initial work with the fill in login boxes and redisigned them to make the interface look cleanerand more readable.
